# Security Policy

The Relational Calculus Foundation takes the security of our software and
community seriously. We appreciate responsible disclosure of vulnerabilities.

## Supported Versions

We currently support the latest release and the `main` branch with security
updates. Older versions may not receive patches.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |
| latest release | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public
issue. Instead, send a detailed report via email to:

**security@relationalcalculus.org** (to be created)

Include the following in your report:

- A clear description of the vulnerability.
- Steps to reproduce or proof-of-concept code.
- The potential impact (e.g., data leakage, code execution).
- Any suggested fixes, if available.

We will acknowledge receipt of your report within **48 hours** and provide an
estimated timeline for a fix. We ask that you keep the vulnerability
confidential until we have released a patch and given users time to update.

## Our Commitment

- We will investigate all legitimate reports and make every effort to correct
  the issue promptly.
- We will credit the reporter in the release notes, unless you prefer to remain
  anonymous.
- We will not take legal action against researchers who follow this responsible
  disclosure policy.

## Third-Party Libraries

While we strive to keep dependencies up to date, vulnerabilities in third-party
packages should be reported to their respective maintainers. If a dependency
poses a direct threat to the Relational Calculus Framework, please let us know
as well so we can issue an advisory.

Thank you for helping keep the Relational Calculus ecosystem safe.
